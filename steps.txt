DOCQUEST - HOW TO RUN THIS PROJECT (Local)
============================================

STEP 1: Open the project folder
- Open terminal (PowerShell) inside the DocQuest_Project folder
- Or: open folder -> click address bar -> type "cmd" -> Enter

STEP 2: Create virtual environment (only needed once, or if venv was deleted)
python -m venv venv

STEP 3: Activate the virtual environment
On Windows PowerShell:
venv\Scripts\activate

You should see (venv) appear at the start of your terminal line - this means it's active.

STEP 4: Install all required packages
pip install -r requirements.txt

STEP 5: Check .env file exists with your API key
Open .env file, confirm it has your Groq API key in this format:
GROQ_API_KEY=your_actual_key_here
(If missing, get a new key from https://console.groq.com and add it here)

STEP 6: Run the app
streamlit run app.py

STEP 7: Open in browser
- It should auto-open, or manually go to the "Local URL" shown in terminal
  (usually http://localhost:8501)
- "Network URL" shown in terminal lets other devices on same WiFi access it too

STEP 8: Using the app
- Upload a PDF using the upload button
- Wait for it to process/index the document
- Type your question in the chat box and hit enter

TO STOP THE APP:
- Go to terminal, press CTRL + C

NEXT TIME YOU OPEN THIS PROJECT:
- You only need Steps 1, 3, 6, 7 (skip 2 and 4 since venv+packages already installed)
- If venv was deleted or packages missing, redo Steps 2-4 first

TROUBLESHOOTING:
- "streamlit not recognized" -> venv not activated, redo Step 3
- "Module not found" -> redo Step 4
- API errors -> check .env key is correct and not expired
